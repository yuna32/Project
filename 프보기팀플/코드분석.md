## 코드 동작 분석

### 1. 데이터 로딩 (VAD 처리)

- `data/` 폴더에서 `.wav` (오디오) 파일을 먼저 찾기
- 짝이 되는 `.csv` (IMU) 파일이 있는지 확인
- `librosa` 라이브러리로 `.wav` 파일을 열어 **침묵 구간을 제거**, 실제 **발화가 시작하고 끝나는 시간**(예: 0.4초~2.1초)을 찾아내기
- `.csv` 파일의 `timestamp`를 이 시작/종료 시간으로 필터링
- 이렇게 '발화 구간'만 남은 IMU 데이터를 100Hz로 리샘플링하고 노이즈를 제거

### 2. 데이터 정제 및 분리

- **레이블 필터링:** VAD 처리 후 샘플이 1개뿐인 발화자(레이블)가 있다면 `stratify` 오류 방지를 위해 해당 데이터를 제거
- **훈련/테스트 분리:** 모든 데이터를 훈련 셋(80%)과 테스트 셋(20%)으로 나눈다. (`stratify` 옵션으로 각 발화자의 데이터가 8:2 비율로 골고루 나뉘도록 보장.)

### 3. 표준화 및 피처 추출

- **표준화:** 모든 훈련 데이터(`X_train`)를 합쳐 6축(ax~gz)의 평균과 표준편차를 계산하고, 이 기준으로 훈련 셋과 테스트 셋 **전부**를 스케일링
- **피처 추출 (핵심):**
    - `(가변 길이, 6)` 형태의 시계열 데이터를 RandomForest가 학습할 수 다.
    - `extract_features` 함수를 사용해 각 샘플을 **고정된 크기 `(30,)`의 벡터로 요약
    - (내용: 6축 각각의 `mean`, `std`, `min`, `max`, `median` 값. 6축 * 5개 피처 = 30개)
    - `X_train_features`는 이제 (훈련 샘플 수, 30) 크기의 2D 배열이 됨

### 4. 모델 훈련 및 평가

- **훈련:** `(훈련 샘플 수, 30)` 크기의 피처 벡터를 `RandomForestClassifier` 모델에 넣어 훈련
- **평가:** 훈련된 모델로 `X_test_features`를 예측
- **결과 출력:** 터미널에 **정확도**와 **분류 리포트**를 텍스트로 출력
- **시각화:** 발화자 ID `1`, `2`, `3`... 등이 얼마나 잘 분류되었는지 보여주는 **Confusion Matrix 그래프**를 팝업 창으로 띄움
